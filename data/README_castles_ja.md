# 日本100名城・続日本100名城 データセット

このリポジトリには、**日本100名城** と **続日本100名城** の計200件を、日本地図上にマッピングするための監査済み JSON データセットが含まれています。

このデータは、Web地図、アプリ、可視化プロジェクトなどで実用的に使えるように、座標・名称・読み・スキーマを手作業で整理し、出典情報を付けて監査したものです。

---

## このデータセットは何か

この JSON ファイルは、日本の200の城郭関連史跡について、以下を含む構造化データです。

- **日本100名城**
- **続日本100名城**

各レコードには、識別情報、日本語名、英語名、読み、位置情報、そしてその座標をどのように確認したかを示す監査メタデータが含まれています。

これは単なる「城名と緯度経度の一覧」ではありません。  
**再利用可能で、出典を追跡できる地理参照データ**として整備したものです。

---

## このデータセットの重要性

一般に公開されている城データには、次のような問題がよくあります。

- 座標が城跡そのものではなく、駅・博物館・駐車場・スタンプ設置場所を指している
- 英語表記に揺れや誤記がある
- 日本語の読みがない
- ID が重複している
- 命名規則が統一されていない
- どの出典に基づいて座標修正したのか分からない

このデータセットは、そうした問題を解消するために作成されました。

最終版では、次の点を重視しています。

- **統一された構造**
- **地図表示に適した代表座標**
- **追加された日本語読み**
- **整理された英語表記**
- **追跡可能な監査メタデータ**
- **200件全件を網羅**

そのため、地図、可視化、教育用途、Webサイト、アプリなどに再利用しやすく、未監査の座標一覧よりもはるかに信頼性の高いデータになっています。

---

## 基本設計方針: `site_center`

このデータセットでもっとも重要な設計方針は、主座標を次で統一したことです。

```json
"coordinate_type": "site_center"
```

### `site_center` とは何か

この `lat` / `lng` は、**その史跡を地図上で自然に示す代表点**を意味します。

主に表したいのは、城郭史跡そのものです。  
したがって、原則として以下は主座標にしていません。

- スタンプ設置場所
- 資料館
- ビジターセンター
- 駐車場
- 最寄駅
- 登城口
- 市役所・観光案内所

### 城のタイプごとの考え方

- **平城・近世城郭**: 本丸・天守台周辺の代表点
- **山城**: 主郭付近の代表点
- **館・居館・城館**: 中心遺構の代表点
- **チャシ・遺跡群**: 主要遺構群の代表点

つまりこのデータは、**観光施設案内用の座標ではなく、地図上で「城がどこにあるか」を示すための代表点データ**です。

## JSON の構造

各オブジェクトは、おおむね次のような構造になっています。

```json
{
  "id": 1,
  "list_type": "100",
  "number": 1,
  "name_ja": "根室半島チャシ跡群",
  "reading_ja": "ねむろはんとうちゃしあとぐん",
  "name_en": "Nemuro Peninsula Chashi Ruins",
  "prefecture": "北海道",
  "prefecture_en": "Hokkaido",
  "tower_status": "ruins",
  "is_national_treasure": false,
  "built_year": null,
  "lat": 43.39075,
  "lng": 145.66838,
  "coordinate_type": "site_center",
  "source_primary": "…",
  "source_secondary": "…",
  "verification_confidence": "high",
  "verification_note": "…"
}
```

---

## フィールド一覧

### 識別情報
- `id`  
  内部ユニークID。重複は解消済みです。

- `list_type`  
  `"100"` または `"continued_100"`。

- `number`  
  名城リスト上の番号。

### 名称
- `name_ja`  
  日本語の主表示名。

- `reading_ja`  
  日本語の読み。ひらがなで統一しています。

- `name_en`  
  正規化した英語表記。

### 位置情報
- `prefecture`  
  都道府県名（日本語）。

- `prefecture_en`  
  都道府県名（英語）。

- `lat`, `lng`  
  地図表示用の主座標。

- `coordinate_type`  
  主座標の意味。最終版ではすべて `"site_center"`。

### 城郭メタデータ
- `tower_status`  
  天守・構造物の状態を表す正規化フィールド。  
  代表的な値:
  - `existing`
  - `reconstructed_concrete`
  - `reconstructed_wood`
  - `replica`
  - `ruins`

- `is_national_treasure`  
  国宝フラグ。

- `built_year`  
  築城年または創建年の代表値。

### 監査メタデータ
- `source_primary`  
  主たる照合ソース。

- `source_secondary`  
  必要に応じた補助ソース。

- `verification_confidence`  
  採用座標の確信度。

- `verification_note`  
  なぜその座標を採用したかの簡単な説明。

## このデータが作られた経緯

このデータセットは、もともとの `castles.json` を出発点にしています。  
元ファイルには有用な基礎データがありましたが、同時にいくつか問題もありました。

### 元ファイルの主な問題
- 一部の座標が不正確
- 英語表記に誤り・不自然さ・揺れがある
- 読みがない、または不十分
- スキーマに一貫性がない
- ID が重複している

### 今回行った作業
このデータは、段階的に以下の手順で改訂されました。

1. **スキーマの正規化**
   - キー名を整理
   - 命名規則を統一
   - `coordinate_type` を追加

2. **名称整備**
   - 日本語名・英語名の不整合を修正
   - ローマ字表記の問題を整理
   - `reading_ja` を追加

3. **座標監査**
   - 全200件を段階的に再照合
   - `site_center` 基準で主座標を統一

4. **監査メタデータ付与**
   - 出典
   - 確信度
   - 採用理由
   を記録

5. **最終監査**
   - 中間ログの不整合を修正
   - `medium` 判定を再点検
   - 最終版では **全200件を `high`** に統一

## 参照したソース

このデータセットは恣意的に座標を置き換えたものではなく、公開情報をもとに照合したものです。

### 主に使ったソース
- **Jcastle**
- **自治体公式ページ**
- **文化遺産オンライン**
- **文化財・史跡系の公的データベース**
- **地理参照データ**
- **地図リンク付きの公式スポット案内**
- 必要に応じて **観光協会公式**
- ごく一部で補助的に一般公開資料

### 基本的な優先順位
1. 自治体・管理団体の公式情報
2. 公的文化財・文化遺産データ
3. Jcastle の個別ページ
4. 地理参照データ
5. 補助的な地図付き参照先

### Jcastle を多用した理由
Jcastle は多くの城について **個別ページに Coordinates が明記**されており、英語名の確認にも使いやすいため、今回の照合作業では非常に重要なソースになりました。

---

## 座標照合の考え方

### 基本ルール
各城について、以下の流れで確認しました。

1. 対象城を特定
2. 日本語名・英語名・読みを確認
3. 城跡本体の場所を確認
4. `site_center` として妥当な代表点を決定
5. `lat` / `lng` を更新
6. 出典と補足を記録
7. 確信度を判定

### 確信度
- `high`  
  信頼できるソースに明示座標がある、または複数ソースで十分裏取りできたもの

- `medium`  
  史跡の同定はできるが、座標がやや推定寄りのもの

最終的には `medium` を再点検し、**最終版では 200件すべて `high`** にしています。

---

## 英語表記について

英語表記は、すべてを無理に `Castle` に統一するのではなく、史跡の性格に応じて整理しました。

### 基本方針
- 一般的な城郭 → `Castle`
- 遺跡性が強いもの → `Ruins`
- 館 → `Yakata` または `Fortified Residence`
- 明らかな誤記や不自然な綴りは修正

### 例
- `Tagajo` → `Taga Castle`
- `Suwaharajiro Castle` → `Suwahara Castle`
- `Sugaya-yakata` → `Sugaya Yakata`
- `Nemuro Peninsula Chashi Site` → `Nemuro Peninsula Chashi Ruins`

このため、厳密に一つの学術英語表記へ固定するというより、**地図データとしての分かりやすさと一貫性**を優先しています。

---

## 日本語読みについて

`reading_ja` は、次の目的のために追加されました。

- 検索しやすくする
- ソートや表示をしやすくする
- 将来的なローマ字変換の補助にする

読みは **ひらがな** で統一しています。

---

## 信頼性と限界

### 強み
- 200件をすべて網羅
- スキーマを整理済み
- 地図用途に適した `site_center` 基準
- 手作業で再照合
- 出典・監査メタデータ付き
- 最終版は全件 `high`

### 限界
- `site_center` は代表点であり、法的な史跡境界中心ではありません
- 城域が非常に広い城は、単一点では表現しきれません
- 英語名はソースによって揺れうります
- 考古学的な境界ポリゴンデータではなく、**地図表示向けの代表点データ**です

つまり、このファイルは **高品質な地図用基準データ**であって、発掘区画や境界測量データではありません。
